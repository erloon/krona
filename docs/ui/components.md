# UI Component Inventory

This document translates the screen prototypes in `docs/ui` into a reusable component inventory for the app. The inventory is based on these prototype folders: `add_income_refined`, `dashboard_with_burden_bar`, `dodaj_koszt_z_walutami`, `income_list_styled_header`, `lista_koszt_w_szczeg_owe_podsumowanie`, `settings_paper_logic_style`, `splash_screen`, and `updated_login_screen`.

Shared UI components should map primarily to `src/shared/ui/primitives` or `src/shared/ui/layout`. Feature components should map primarily to `src/features/calculator/presentation/components`.

## Shared UI Components

- `AppTopBar` Reusable top navigation bar with a title, optional back or menu action, and right-side actions or avatar. It should cover dashboard, lists, settings, and form flows without embedding screen-specific logic. Reference: `docs/ui/add_income_refined/code.html`, `docs/ui/dashboard_with_burden_bar/code.html`, `docs/ui/dodaj_koszt_z_walutami/code.html`, `docs/ui/income_list_styled_header/code.html`, `docs/ui/lista_koszt_w_szczeg_owe_podsumowanie/code.html`, `docs/ui/settings_paper_logic_style/code.html`.
- `BottomTabBar` Persistent bottom navigation with icon and label items plus active-tab state. It should be shared by dashboard, income, cost, and settings screens so navigation stays consistent. Reference: `docs/ui/dashboard_with_burden_bar/code.html`, `docs/ui/income_list_styled_header/code.html`, `docs/ui/lista_koszt_w_szczeg_owe_podsumowanie/code.html`.
- `ScreenContainer` Base page wrapper that handles max width, horizontal padding, vertical spacing, and safe-area offsets. It keeps route files thin and gives every screen the same structural rhythm. Reference: all `docs/ui/*/code.html` screens.
- `SectionHeader` Small heading block for section title, optional subtitle, and optional trailing action. It should be reusable across list sections, settings groups, and breakdown areas. Reference: `docs/ui/dashboard_with_burden_bar/code.html`, `docs/ui/income_list_styled_header/code.html`, `docs/ui/lista_koszt_w_szczeg_owe_podsumowanie/code.html`, `docs/ui/settings_paper_logic_style/code.html`.
- `PrimaryButton` High-emphasis action button for save, add, and submit flows. It should support icon content, full-width layout, and standard loading or disabled states. Reference: `docs/ui/add_income_refined/code.html`, `docs/ui/dashboard_with_burden_bar/code.html`, `docs/ui/dodaj_koszt_z_walutami/code.html`.
- `SecondaryButton` Lower-emphasis action button for edit, duplicate, filter, and cancel actions. It should be reusable inside cards, forms, and compact action rows. Reference: `docs/ui/dodaj_koszt_z_walutami/code.html`, `docs/ui/income_list_styled_header/code.html`, `docs/ui/lista_koszt_w_szczeg_owe_podsumowanie/code.html`.
- `IconButton` Compact tap target for menu, delete, filter, calendar, search, and back actions. It keeps icon-only interactions consistent across the app shell and content areas. Reference: `docs/ui/add_income_refined/code.html`, `docs/ui/dashboard_with_burden_bar/code.html`, `docs/ui/dodaj_koszt_z_walutami/code.html`, `docs/ui/income_list_styled_header/code.html`, `docs/ui/lista_koszt_w_szczeg_owe_podsumowanie/code.html`, `docs/ui/settings_paper_logic_style/code.html`.
- `FloatingActionButton` Floating add button used on list screens for primary creation flows. It should support icon-only content and the elevated visual treatment used in the prototypes. Reference: `docs/ui/income_list_styled_header/code.html`, `docs/ui/lista_koszt_w_szczeg_owe_podsumowanie/code.html`.
- `TextField` Standard single-line input with label, placeholder, optional suffix, and consistent focus styling. It covers fields such as company name, invoice number, client name, and search-adjacent entry. Reference: `docs/ui/add_income_refined/code.html`, `docs/ui/dodaj_koszt_z_walutami/code.html`, `docs/ui/settings_paper_logic_style/code.html`.
- `TextAreaField` Multi-line input for longer content such as descriptions and addresses. It should share the same labeling and state rules as `TextField` while allowing larger editable content. Reference: `docs/ui/add_income_refined/code.html`, `docs/ui/settings_paper_logic_style/code.html`.
- `NumericAmountField` Large-format numeric input for monetary values with a visible currency suffix. It is the base amount entry pattern for both income and cost forms. Reference: `docs/ui/add_income_refined/code.html`, `docs/ui/dodaj_koszt_z_walutami/code.html`.
- `SearchField` Search-specific input with leading icon and compact height. It should be optimized for list filtering without becoming a separate one-off input style. Reference: `docs/ui/income_list_styled_header/code.html`.
- `SelectField` Reusable dropdown control for settings flows such as ryczałt rate selection. It should follow the same field structure and label conventions as the text inputs. Reference: `docs/ui/settings_paper_logic_style/code.html`.
- `ToggleSwitch` Binary on or off control for settings like voluntary sickness insurance and tax relief flags. It must expose checked, unchecked, and disabled states with optional descriptive text. Reference: `docs/ui/settings_paper_logic_style/code.html`.
- `ChoiceCard` Selectable row card with title, description, icon, and checked state. It should be the common base for tax form selection, settlement type selection, deduction mode, and ZUS status. Reference: `docs/ui/add_income_refined/code.html`, `docs/ui/dodaj_koszt_z_walutami/code.html`, `docs/ui/settings_paper_logic_style/code.html`.
- `ChipSelector` Horizontal or wrapped group of selectable chips for currencies, VAT rates, and similar short options. It should support single-select behavior and clear visual distinction for the active choice. Reference: `docs/ui/add_income_refined/code.html`, `docs/ui/dodaj_koszt_z_walutami/code.html`.
- `StatusTag` Small contextual badge for values like VAT rate, deduction mode, or cost type. It is intended for use inside cards, summaries, and small metadata clusters. Reference: `docs/ui/income_list_styled_header/code.html`, `docs/ui/lista_koszt_w_szczeg_owe_podsumowanie/code.html`.
- `InfoBanner` Compact inline information strip with icon and supportive secondary text. It is useful for deduction summaries, helper notes, and lightweight contextual guidance. Reference: `docs/ui/lista_koszt_w_szczeg_owe_podsumowanie/code.html`.
- `MetricCard` Reusable card for one financial metric such as income, cost, VAT, or PIT. It should support a label, amount, currency, and optional accent emphasis without being tied to one screen. Reference: `docs/ui/dashboard_with_burden_bar/code.html`.
- `ProgressBar` Simple progress or ratio bar for burden and threshold visualizations. It should work for both single-fill indicators and segmented states where needed. Reference: `docs/ui/dashboard_with_burden_bar/code.html`, `docs/ui/settings_paper_logic_style/code.html`, `docs/ui/splash_screen/code.html`.
- `SummaryHero` Large headline summary block for the primary number on a screen. It should serve dashboard totals and list summary headers without carrying feature-specific calculations. Reference: `docs/ui/dashboard_with_burden_bar/code.html`, `docs/ui/income_list_styled_header/code.html`, `docs/ui/lista_koszt_w_szczeg_owe_podsumowanie/code.html`.
- `EmptyState` Generic placeholder for screens with no records yet. It is not shown directly in the prototypes, but the real app will need a reusable empty-state pattern for lists and sections. Reference: inferred from list screens `docs/ui/income_list_styled_header/code.html` and `docs/ui/lista_koszt_w_szczeg_owe_podsumowanie/code.html`.
- `Avatar` Small circular profile image used in top bars and settings areas. It should stay presentation-only and avoid coupling to user-fetching logic. Reference: `docs/ui/dashboard_with_burden_bar/code.html`, `docs/ui/income_list_styled_header/code.html`, `docs/ui/lista_koszt_w_szczeg_owe_podsumowanie/code.html`, `docs/ui/settings_paper_logic_style/code.html`.
- `BrandMark` Compact brand icon and wordmark lockup used on splash and login surfaces. It should be reusable across authentication and startup flows without navigation shell dependencies. Reference: `docs/ui/splash_screen/code.html`, `docs/ui/updated_login_screen/code.html`.
- `LoadingIndicator` Minimal branded loading indicator for splash and async transitions. It should support a short label so the app can communicate startup or loading status consistently. Reference: `docs/ui/splash_screen/code.html`.

## Feature Components

- `FinancialBreakdownRow` Row showing a tax or contribution label with its amount. It is the smallest reusable unit inside detailed tax and deduction breakdowns. Reference: `docs/ui/dashboard_with_burden_bar/code.html`.
- `FinancialBreakdownCard` Grouped or expandable card for taxes and contributions with a summary total and detailed rows. It is based on the dashboard breakdown pattern and belongs to calculator-specific presentation. Reference: `docs/ui/dashboard_with_burden_bar/code.html`.
- `IncomeSummaryHeader` Editorial-style summary block with total net income plus VAT and PIT sub-metrics. It is tailored to the income listing experience and should compose shared summary primitives. Reference: `docs/ui/income_list_styled_header/code.html`.
- `CostSummaryHeader` Equivalent summary block for costs with total net amount plus VAT and PIT sub-metrics. It mirrors the income pattern while staying specific to cost flows. Reference: `docs/ui/lista_koszt_w_szczeg_owe_podsumowanie/code.html`.
- `IncomeListItemCard` Reusable income record card with client or title, invoice metadata, amount, VAT tag, and actions. It drives the presentation of income entries in list screens. Reference: `docs/ui/income_list_styled_header/code.html`.
- `CostListItemCard` Reusable cost record card with category tag, title, amount breakdown, deduction info, and actions. It is specific to cost entries because it carries deduction-related metadata. Reference: `docs/ui/lista_koszt_w_szczeg_owe_podsumowanie/code.html`.
- `RecordActionRow` Shared action cluster for edit, duplicate, and delete operations on list items. It can be reused by both income and cost cards while staying within calculator presentation. Reference: `docs/ui/income_list_styled_header/code.html`, `docs/ui/lista_koszt_w_szczeg_owe_podsumowanie/code.html`.
- `AmountSummaryPanel` Form-side summary block showing gross, net, VAT, and selected currency. It is especially important on add income flows where totals react to form input. Reference: `docs/ui/add_income_refined/code.html`.
- `SettlementTypeSelector` Specialized selector for monthly, daily, and hourly income settlement modes. It should be built on top of `ChoiceCard`, but remain feature-specific because the options encode calculator behavior. Reference: `docs/ui/add_income_refined/code.html`.
- `WorkParametersSection` Paired numeric inputs for work days and hours. It belongs to the calculator feature because it only appears when time-based revenue modes are active. Reference: `docs/ui/add_income_refined/code.html`.
- `DeductionModeSelector` Specialized selector for standard and vehicle-related deduction modes. It should compose `ChoiceCard` while keeping the deduction-domain options inside the feature layer. Reference: `docs/ui/dodaj_koszt_z_walutami/code.html`.
- `AttachmentDropzone` Receipt or invoice attachment area with icon and helper text. It is specific to cost creation and editing workflows where document capture matters. Reference: `docs/ui/dodaj_koszt_z_walutami/code.html`.
- `SettingsGroupCard` Grouped settings container with spacing and hierarchy for profile, tax, ZUS, and relief configuration. It keeps the settings screen modular while remaining tied to calculator settings content. Reference: `docs/ui/settings_paper_logic_style/code.html`.
- `ThresholdIndicator` Segmented progress display for health-insurance thresholds. It comes from the settings prototype and represents calculator-specific policy information rather than a generic progress bar. Reference: `docs/ui/settings_paper_logic_style/code.html`.
- `ReliefToggleRow` Settings row combining relief title, description, and toggle, with optional nested numeric input. It is designed for IP Box and similar tax relief configuration. Reference: `docs/ui/settings_paper_logic_style/code.html`.
- `AuthCard` Lightweight authentication surface containing logo, heading, explanatory text, and provider actions. It is feature-level because it composes shared branding and button primitives into a login experience. Reference: `docs/ui/updated_login_screen/code.html`.
- `GoogleSignInButton` Provider-specific sign-in action with brand icon and standardized label. It can stay in feature presentation unless the app later adds a broader auth provider system. Reference: `docs/ui/updated_login_screen/code.html`.
- `SplashBrandScreen` Full-screen branded startup layout composed from `BrandMark` and `LoadingIndicator`. It is intentionally screen-scoped because it suppresses the normal app shell and exists only for app startup. Reference: `docs/ui/splash_screen/code.html`.
